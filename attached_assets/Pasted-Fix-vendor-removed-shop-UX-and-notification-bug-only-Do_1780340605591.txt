Problem:
When admin removes/deletes a vendor shop:
- The shop is removed from customer side.
- Vendor product upload stops.

     type: "warning"

2. When admin bans/rejects shop:
   - Update related user:
     vendorStatus = "rejected"
     role = "customer"

   - After shop delete/ban, /api/auth/me must return updated role/vendorStatus.
   - VendorGuard must block vendor dashboard if vendorStatus !== "approved".
   - If user is no longer approved vendor, redirect to VendorStatus or vendor-register.

5. Notification center:
   - Removed vendor should see the admin removal notification in notifications page.
   - Header notification badge should update.

6. Test:

### 3. Images Audit

| Check | Status | Findings | Recommendation |
| :--- | :---: | :--- | :--- |

### Top 10 Fixes (Priority Order)

1.  **Register Missing Routes in `App.tsx` (High)**
    Add routes and page components for `/privacy`, `/terms`, `/refund-cancellation`, and `/contact-support` to prevent soft 404s for URLs listed in the sitemap.
2.  **Add a Header (H1) Tag on Key Pages (High)**
    Insert a single `<h1>` tag on the Home page, Shops list, and Search page to establish the crawl outline.
3.  **Implement a Site Footer with Internal Links (High)**
    Build a global `<Footer>` component containing links to all missing routes to eliminate orphan pages.
4.  **Convert Images to WebP Format (Medium)**
    Replace all category and product PNG images with WebP to improve page speed and Core Web Vitals.
5.  **Enable Lazy Loading for Images (Medium)**
    Add `loading="lazy"` to all image tags inside `ProductCard` and `CategoryBubble` components to optimize rendering.
6.  **Add Dynamic Canonical URL Logic (Medium)**
    Fix the hardcoded canonical in `index.html` by making it output the exact path requested by crawlers.
7.  **Add FAQ Schema to Homepage (Medium)**
    Implement a collapsable FAQ accordion on the Home page to satisfy search engine query requirements.
8.  **Define Width and Height on Images (Medium)**
    Add explicit size attributes on icons and cards to prevent layout shifts.
9.  **Write Unique Descriptions & Image Names for Products (Low)**
    Stop sharing category placeholder images; use unique product images named after the product keyword.
10. **Implement Breadcrumbs (Low)**
    Add breadcrumbs to category and product pages (e.g., `/category/groceries` -> `/product/g1`) to improve internal linking architecture.

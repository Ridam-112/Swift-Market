Push the DB schema to Neon (run in Shell):

DATABASE_URL="$NEON_DATABASE_URL" pnpm --filter @workspace/db run push

Start the app — run the Start application workflow (or artifacts/api-server: API Server + artifacts/swiftmart: web workflows separately).

The app uses the Neon database (NEON_DATABASE_URL takes priority over Replit's built-in DATABASE_URL), so all existing shops, products, users, and orders will be visible immediately.

OTP login is in demo mode — use code 123456 for any phone number during testing. Set OTP_MODE=real in environment variables when ready for production SMS.

Super admin phones are set in the SUPER_ADMIN_PHONES environment variable (already configured in .replit).

⚠️ Important: The NEON_DATABASE_URL above contains credentials — treat it as a secret and never commit it to code. Only paste it into Replit Secrets.

ProductsContext and ShopsContext fire GET /api/products and GET /api/shops every 5 seconds unconditionally, even when the user is idle or on an unrelated page. Browser console confirmed 2 API calls every 5 s non-stop.		 ProductsContext.tsx:89
ShopsContext.tsx:98		 Wastes ~24 API calls/min per user. Hammers the DB. Kills battery on mobile. Inflates server costs.
Critical Database No DB-level referential integrity
Foreign keys (shopId in products, customerId in orders, userId in deliveryPartners) are plain text columns with no Drizzle references(). Deleting a shop leaves orphaned products, orders, and notifications with no cascade.		 lib/db/src/schema.ts Silent data corruption. Orphaned rows accumulate indefinitely. Joins return stale/ghost data.

Medium Frontend Race condition — Product.tsx async fetch
Product detail fetches product by ID without a cleanup/abort flag. If the route changes while a request is in flight, a stale response can overwrite the new product state.		 Product.tsx:40 Wrong product shown briefly. Could confuse add-to-cart with the wrong item.
